## Design Context

### Users
Mariano Mattei — solo real estate investor + AI/cybersecurity professional. Uses this dashboard as his personal ops terminal while building a BRRRR/STR portfolio. Typical session: late night, MacBook, dark office, reviewing deal flow and tax positioning before making capital commits. High-stakes financial decisions, needs data density and clarity over decoration.

### Brand Personality
**Tactical. Dense. Copper-edged.** Three words: *precision, weight, momentum*.

Emotional goal: feels like a specialized trading terminal crossed with a construction ops center. Not a SaaS product dashboard. Not a pretty startup landing page. A real tool a serious operator actually runs.

Anti-reference: anything that looks like an AI-generated fintech dashboard — card grids with hero metrics, gradient text, glassmorphism, neon accents, sparkline decorations.

### Aesthetic Direction
- **Dark always** — user is working at night reviewing deals, never in a bright office context
- **Copper (#b45309) on deep slate (#0b1220)** — the existing broker theme palette; this is correct and intentional
- **Typography**: Outfit + JetBrains Mono currently in use. Functional for a v1. Consider upgrading display/heading font to something with more editorial weight in v2.
- **Data density over whitespace** — this is an ops dashboard, not a marketing page. Tight layouts are correct.
- **No decorative flourishes** — motion should be reserved for state transitions (deal card moves in kanban, status updates). Nothing animated for decoration.

### Design Principles
1. **Every pixel earns its place** — if an element doesn't convey data or enable action, remove it
2. **Copper accent = signal, not decoration** — use the accent color only where it points to something actionable or high-priority
3. **Keyboard-first data navigation** — this is a power user's terminal; tab, arrow keys, and enter should work everywhere
4. **Inline styles are debt** — all styling should use CSS classes and design tokens, not element.style.cssText scattered throughout
5. **No border-left stripes** — the absolute ban exists for good reason; every instance in this codebase needs redesigning as a background tint or leading badge
